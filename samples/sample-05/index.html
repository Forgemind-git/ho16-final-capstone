Customer Support Co-Pilot

Week 6 M19 · Deployed AI Solution Peer-reviewed Capstone · Sample 05
Support agents spend hours writing replies from scratch, often missing relevant knowledge base articles, while managers have no visibility into ticket trends. This capstone builds an AI-powered support hub where Claude triages tickets, drafts grounded replies from your knowledge base, and produces a trends dashboard — all using Claude.ai subscription features with no API key required.
No API key required
  • No API key required. This solution is built entirely on your Claude.ai Pro or Team subscription — using Projects, file uploads, Artifacts, Skills, and Cowork collaboration.

Architecture — Claude.ai Features Map

Every component of the Support Co-Pilot maps to a Claude.ai subscription feature. Your knowledge base lives in the Project. Your agents use Cowork. Your manager views the Artifact dashboard.

Project

Support Workspace

One Claude.ai Project is your team's shared support centre. All KB articles, triage rules, past tickets, and reply templates are stored here persistently.

File Upload

Knowledge Base

Upload your product documentation, FAQs, refund policies, and troubleshooting guides. Claude reads them as shared Project knowledge — every reply it drafts cites the exact article.

Skill

Support Agent Persona

Project Instructions define Claude as a Senior Support Specialist — with your company name, tone rules, escalation paths, and priority taxonomy hard-coded into every response.

Artifact

Ticket Trends Dashboard

Claude generates an interactive HTML Artifact — a manager dashboard showing ticket volume by category, priority distribution, response time targets, and trend charts over time.

Cowork

Agent Team Room

Invite your support agents to the Project. Each agent works in their own conversation, but all share the same KB, persona instructions, and triage rules — consistent quality across the team.

MCP

Live Order / Account Lookup

Connect a data MCP tool to your order management system. When a customer mentions an order number, Claude looks it up and includes real order status in the draft reply.

Build It Step by Step

  1. Create the Support Project Go to claude.ai → Projects → New Project. Name it "Customer Support Co-Pilot – [Your Company]". This is your team's shared support workspace.
  2. Upload Your Knowledge Base Upload all your support documentation: product guides, FAQ PDFs, refund and returns policies, troubleshooting steps, known issues log. Upload each as a separate file — Claude can search across all of them simultaneously. The more you upload, the better the grounded replies.
  3. Set the Support Agent System Prompt In Project Settings → Instructions, paste the Support Agent prompt (see below). Customise it with your company name, tone (e.g. formal vs. friendly), and your priority taxonomy. Every agent on the team will automatically use this when they work in the Project.
  4. Triage Your First Ticket Start a conversation in the Project. Paste a support ticket (subject + body) and the Triage Prompt. Claude categorises the ticket (Billing / Technical / Returns / General), assigns a priority (Low / Medium / High / Critical), and gives a one-sentence routing recommendation — all in under 3 seconds.
  5. Generate a Grounded Reply Draft After triaging, paste the Reply Draft Prompt. Claude writes a complete customer reply that: (a) opens with empathy, (b) answers the issue using exact steps from the KB articles, (c) cites which articles it used, (d) closes with next steps. Agents edit and send — never start from a blank page.
  6. Process a Batch of Tickets Paste 5–10 tickets at once into a conversation using the Batch Triage Prompt. Claude triages all of them in one response — returning a markdown table with category, priority, and routing for each. Use this at the start of each shift to instantly prioritise the queue.
  7. Generate the Manager Trends Dashboard Artifact Paste the Dashboard Artifact Prompt (listing your recent ticket categories and volumes). Claude creates an interactive HTML Artifact with: category breakdown chart, priority distribution, volume trend over time, and a "top issues this week" section — ready to share with your manager.
  8. Invite Your Support Team (Cowork) In Project Settings → Members, invite your support agents. Each agent works in their own conversation thread but inherits the same KB uploads and system prompt. Managers can browse all conversations to QA replies and spot training gaps.

Prompts & Configs

Project Instructions (System Prompt)
You are a Senior Customer Support Specialist for [YOUR COMPANY NAME].

Your responsibilities:
1. TRIAGE tickets — classify by category and assign priority:
   Categories: Billing | Technical | Returns & Refunds | Account | General Inquiry
   Priorities: Critical (SLA 1hr) | High (SLA 4hr) | Medium (SLA 24hr) | Low (SLA 72hr)

2. DRAFT REPLIES — write complete, ready-to-send customer replies:
   - Always open with empathy (1 sentence acknowledging the frustration)
   - Answer using ONLY information from the uploaded KB articles
   - Cite which article(s) you used: "According to our [Article Name]..."
   - Close with clear next steps and a timeframe
   - Tone: [PROFESSIONAL / FRIENDLY / FORMAL — pick one]
   - Sign off as: "The [Company] Support Team"

3. NEVER promise a specific outcome (refund amount, fix date) unless it's explicitly stated in the KB.
   Use "We will review..." / "Our team will..." for anything requiring internal action.

4. ESCALATE if: the ticket mentions legal action, press, social media threats, or data breach. Flag with 🚨 ESCALATE and do not draft a reply.

Ticket priority rule: if a customer uses words like "urgent", "ASAP", "immediately", bump priority one level higher.
Single Ticket Triage Prompt
Triage this support ticket:

Subject: [TICKET SUBJECT]
Customer Message:
[PASTE FULL TICKET BODY HERE]

Return:
- Category: [Billing / Technical / Returns & Refunds / Account / General Inquiry]
- Priority: [Critical / High / Medium / Low]
- SLA Target: [1hr / 4hr / 24hr / 72hr]
- Routing: [team or person this should go to]
- Summary: one sentence describing what the customer needs
- Escalation flag: 🚨 ESCALATE if applicable, otherwise "None"
Grounded Reply Draft Prompt
Draft a customer reply for this ticket:

Subject: [TICKET SUBJECT]
Customer Message:
[PASTE TICKET BODY]

Requirements:
- Open with empathy (1 sentence)
- Provide the complete answer using ONLY the KB articles uploaded to this Project
- Cite which KB article(s) you're drawing from: "As outlined in our [Article Title]..."
- Include step-by-step instructions if troubleshooting is needed
- Close with: what happens next + expected timeframe
- Sign off: "The [Company Name] Support Team"
- Flag any part of the answer where the KB doesn't provide enough information: [KB GAP: ...]

Do NOT add any information not found in the KB.
Batch Ticket Triage Prompt
Triage the following [N] support tickets. Return a markdown table.

Table columns: # | Subject (truncated) | Category | Priority | SLA | Routing | Escalate?

Sort by priority (Critical first, Low last).

Tickets:
---
1. Subject: [SUBJECT 1]
   Message: [BODY 1]
---
2. Subject: [SUBJECT 2]
   Message: [BODY 2]
---
[continue for all tickets]

After the table, flag any 🚨 ESCALATE tickets with a brief reason.
Ticket Trends Dashboard Artifact Prompt
Generate a self-contained HTML Artifact: a Customer Support Manager Dashboard.

Use this ticket data from the past [TIME PERIOD]:
[PASTE SUMMARY — e.g. "Billing: 45 tickets, Technical: 67, Returns: 23, Account: 12, General: 8"]
[Priority breakdown: Critical: 8, High: 34, Medium: 72, Low: 41]
[Daily volume for past 7 days: Mon:22, Tue:18, Wed:31, Thu:27, Fri:35, Sat:12, Sun:10]

Dashboard sections:
1. Header: company name, report period, total tickets, avg daily volume
2. Category breakdown: horizontal bar chart (use inline SVG or canvas)
3. Priority distribution: donut or pie chart
4. Volume trend: 7-day line chart
5. Top issues this week: bullet list of 5 most common complaint types (infer from categories)
6. SLA compliance: % of tickets responded within SLA (estimate from data)

Styling: #1E40AF primary, clean card layout, professional enough to share with executives.
Fully self-contained HTML — no fetch() calls, no external CDN.
Weekly Trend Summary Prompt
Based on the ticket data in this conversation, write a Weekly Support Trends Summary for my manager.

Format:
**WEEK OF:** [date range]
**TOTAL TICKETS:** [N] ([+/-N%] vs last week)

**TOP 3 TICKET CATEGORIES:**
1. [Category] — [N] tickets ([%]) — [one-line insight]
2. ...
3. ...

**EMERGING ISSUE:** [any category trending upward — flag with ⚠️ if >20% WoW increase]

**SLA PERFORMANCE:** [% within SLA] — [On Target / At Risk / Breached]

**KNOWLEDGE BASE GAPS IDENTIFIED:** [list any KB GAP flags from reply drafts this week]

**RECOMMENDATION:** [one concrete action — e.g. "Add an FAQ article on X" or "Route Y tickets to Z team"]

Keep it under 200 words. Suitable for a Slack message or email update.

Done When

  • Support Project created with company-specific system prompt and tone configured
  • Knowledge base uploaded — minimum 3 articles (FAQ, refund policy, troubleshooting guide)
  • At least 5 tickets individually triaged with correct category, priority, and SLA
  • At least 3 grounded reply drafts generated — each citing specific KB articles
  • KB gaps identified and flagged in at least 1 reply (where KB doesn't cover the issue)
  • Batch triage run on 10 tickets at once — sorted markdown table produced correctly
  • Manager Trends Dashboard Artifact generated with working charts
  • Weekly Trend Summary generated and suitable for a manager Slack message
  • Support team invited to Project — at least 1 agent drafted a reply using the shared KB
  • Demo: paste a ticket → triage → draft reply → show KB citation — end-to-end in 60 seconds
