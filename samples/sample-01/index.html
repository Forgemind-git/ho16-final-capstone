AI Ops Command Center

Operations teams waste hours manually reviewing documents, status updates, and scattered data. This capstone builds an intelligent ops hub that summarises uploaded documents, answers grounded questions, and tracks key metrics — all powered by Claude.ai with no backend and no API key.

No API key required. This solution is built entirely on your Claude.ai Pro or Team subscription — using Projects, Artifacts, file uploads, and MCP connections.

Architecture — Claude.ai Features Map

Each component of the Ops Command Center maps to a Claude.ai subscription feature. No server, no SDK, no infrastructure needed.

Project

Persistent Memory

Create a Claude.ai Project named "Ops Command Center". All uploaded docs and conversation history persist automatically within the Project.

File Upload

Document Ingestion

Upload ops documents (PDFs, Word docs, TXT) directly into the Project. Claude reads and stores them as shared knowledge for all conversations.

Artifact

Live Dashboard

Claude generates a React/HTML Artifact — a live metrics dashboard showing doc count, query count, and last refresh time that runs in your browser.

Skill

System Prompt

Set a Project-level system prompt (Skill/Instructions) that instructs Claude to act as an Ops Assistant, cite sources, and always return structured JSON summaries.

MCP

Live Data Refresh

Connect an MCP data source (e.g. Google Sheets or Notion via Claude.ai integrations) so Claude can pull live operational metrics on demand.

Cowork

Team Q&A

Share the Project with your operations team. Multiple members can ask grounded questions and get consistent AI-powered answers from the same doc set.

Build It Step by Step

  1. Create the Project Go to claude.ai → Projects → New Project. Name it "AI Ops Command Center". Add a description: "Central hub for operations document analysis and Q&A."
  2. Set the System Prompt (Instructions) In Project Settings → Instructions, paste the Ops Assistant system prompt (see Prompts & Configs below). This makes every conversation in the project context-aware.
  3. Upload Your Documents Inside the Project, click "Add to Project" → upload your ops docs (status reports, runbooks, meeting notes, SOPs). These become shared knowledge — every conversation can reference them.
  4. Request Document Summaries Start a new conversation in the Project. Ask Claude to summarise each uploaded document using the Summary Prompt below. Claude will extract key points and action items from the actual content.
  5. Generate the Dashboard Artifact Paste the Dashboard Generation Prompt. Claude creates a live React/HTML Artifact — an interactive metrics panel showing document count, recent summaries, and action items. Click "Use Artifact" to run it in your browser.
  6. Enable Live Data via MCP In Claude.ai Settings → Integrations, connect Google Sheets or Notion. Ask Claude to pull the latest ops metrics from your connected data source. Claude will fetch and display live data in the dashboard Artifact.
  7. Invite Your Team (Cowork) In Project Settings → Members, invite your operations team. Everyone shares the same documents, instructions, and history. Each member can run their own Q&A sessions grounded in the shared docs.
  8. Ask Grounded Questions Test the command center: ask questions like "What are the top 3 action items from this week's status reports?" Claude answers using the uploaded documents as its source of truth.

Prompts & Configs

Project Instructions (System Prompt)
You are an AI Operations Assistant for this team's Command Center.

Your job:
1. Summarise uploaded documents — extract key points, action items, owners, and deadlines.
2. Answer questions grounded ONLY in the uploaded documents. Always cite which document your answer comes from.
3. When asked to generate a dashboard, create a clean HTML/React Artifact showing: total docs, total action items, and a list of recent summaries.
4. Format all summaries as JSON:
   { "title": "...", "key_points": [...], "action_items": [...], "owners": [...] }

Never hallucinate facts. If the answer is not in the documents, say so clearly.
Document Summary Prompt
Please summarise the uploaded document "[DOCUMENT NAME]".

Return a structured JSON block with:
- title: document title or inferred title
- key_points: array of 3–5 main findings or updates
- action_items: array of specific tasks that need to be done
- owners: any named people or teams responsible for action items
- deadline_hints: any dates or time references mentioned

Then write a 2-sentence plain-English summary suitable for a management briefing.
Dashboard Generation Prompt
Generate a self-contained HTML Artifact for an "AI Ops Command Center" dashboard.

Include:
- A header with the title and today's date
- A metrics grid: total documents uploaded, total action items, number of open items
- A "Recent Summaries" section listing each document title + its top action item
- A "Team Activity" section showing who asked what (mock 3 example entries if needed)
- Clean, professional styling using #1E40AF as the primary colour
- Fully static — no fetch() calls needed; embed the data directly

Use the summaries I've generated in this conversation as the source data.
Grounded Q&A Prompt
Using ONLY the documents uploaded to this Project, answer the following:

Question: [YOUR QUESTION HERE]

Requirements:
- Cite the specific document and section your answer comes from
- If multiple documents are relevant, synthesise them and cite each
- If the answer cannot be found in the documents, say "I could not find this in the uploaded documents"
- End with: "Confidence: High / Medium / Low" based on how directly the docs address the question
Daily Ops Digest Prompt
Generate a Daily Ops Digest based on all documents in this Project.

Format:
**DATE:** [today's date]
**SUMMARY:** 2-sentence overview of the current operational state

**TOP ACTION ITEMS (this week):**
1. [action] — Owner: [name] — Due: [date or "ASAP"]
2. ...

**RISKS / BLOCKERS:**
- [any risks or blockers mentioned in any document]

**UPCOMING DEADLINES:**
- [chronological list]

Keep it under 300 words. Make it suitable for a morning standup briefing.

  • At least 3 documents uploaded and summarised with JSON output

  • At least 2 grounded Q&A questions answered with document citations
  • Daily Ops Digest generated successfully
  • At least 1 team member invited to the Project (Cowork)
