Content Production Engine

Week 6 M19 · Deployed AI Solution Peer-reviewed Capstone · Sample 03
Content teams spend days researching topics, writing long-form posts, then manually reformatting the same content for LinkedIn, email, and Twitter. This capstone automates the entire pipeline — research brief to published draft to three social variants — all inside Claude.ai with no API key, no backend, no infrastructure.
No API key required
  • No API key required. This solution is built entirely on your Claude.ai Pro or Team subscription — using Projects, Artifacts, file uploads, Skills, and MCP integrations.

Architecture — Claude.ai Features Map

Every stage of the content pipeline maps to a Claude.ai feature. One subscription. Zero extra tools.

Project

Content Workspace

A single Claude.ai Project stores your brand voice, style guide, all research briefs, drafted posts, and social variants — persistent across every session.

Skill

Brand Voice Instructions

Project Instructions define your brand tone, target audience, writing style rules, and banned phrases. Every piece Claude produces matches your voice automatically.

MCP

Connect a web search MCP so Claude pulls real statistics, current events, and expert quotes into your research brief before writing — no more stale or fabricated data.

Claude generates an interactive HTML Artifact — a content calendar showing all posts by status (Draft / Review / Published) with the ability to mark items as done.

File Upload

Reference Material

Upload brand guidelines, past top-performing posts, or competitor analysis PDFs. Claude learns your style patterns and can avoid repeating previously published angles.

Cowork

Editorial Team

Invite writers, editors, and social managers. Writers draft in the Project; editors review and suggest revisions in the same conversation thread; social managers pull the variants.

Build It Step by Step

  1. Create the Project Go to claude.ai → Projects → New Project. Name it "Content Production Engine". This is your permanent editorial workspace.
  2. Upload Brand Voice Guidelines Upload your brand style guide (PDF or paste it as text). If you don't have one, paste a sample of your best-performing content. Claude will use these as its style reference for every piece it writes.
  3. Set the Content Strategist System Prompt In Project Settings → Instructions, paste the Content Production Engine prompt (see below). Claude now operates as a senior content strategist who always researches before writing and formats output for repurposing.
  4. Generate a Research Brief Paste the Research Brief Prompt with your topic. Claude produces a structured brief: 5 angles, 3 key insights, supporting data points, an outline, and a hook sentence. If MCP web search is connected, it will pull live data.
  5. Draft the Long-Form Post Using the brief Claude just generated, paste the Long-Form Draft Prompt. Claude writes a complete LinkedIn/blog post — introduction, body with subheadings, key takeaways, and a strong CTA — matching your brand voice from the uploaded guidelines.
  6. Repurpose Into 3 Social Variants Paste the Repurpose Prompt referencing the long-form post. Claude produces three distinct formats in one response: a LinkedIn teaser (hook + 5 bullets + CTA), a plain-text email newsletter snippet (200 words), and a Twitter/X thread (5–7 numbered tweets).
  7. Generate the Content Calendar Artifact Paste the Content Calendar Prompt listing your upcoming topics. Claude creates a live HTML Artifact — a visual calendar where you can mark posts as Draft / Review / Published and see your content pipeline at a glance.
  8. Invite Your Editorial Team (Cowork) Share the Project with writers, editors, and social managers. Each team member can request revisions, add comments, or ask Claude to adjust tone — all within the same Project context.

Prompts & Configs

Project Instructions (System Prompt)
You are a Senior Content Strategist and Writer for this brand.

Operating rules:
1. RESEARCH before writing — always generate a structured brief with angles, data points, and an outline before drafting long-form content.
2. MATCH BRAND VOICE — use the style guide uploaded to this Project. If in doubt, write in a direct, clear, authoritative tone. No buzzwords. No jargon.
3. DRAFT for repurposing — always write the long-form post in a way that makes it easy to extract a LinkedIn teaser, email snippet, and Twitter thread.
4. TRACK status — label each piece of content with one of: [BRIEF] [DRAFT] [REVIEW] [PUBLISHED].
5. NEVER fabricate statistics or quotes — if you don't have real data and web search is not available, use "[STAT NEEDED]" as a placeholder.

Content formats you produce:
- Research briefs (structured markdown)
- Long-form LinkedIn/blog posts (600–1200 words)
- LinkedIn teasers (3–5 sentence hook + bullet list)
- Email newsletter snippets (150–250 words)
- Twitter/X threads (5–8 numbered tweets)
Research Brief Prompt
Generate a research brief for this content topic:

Topic: [YOUR TOPIC]
Target Audience: [WHO WILL READ THIS]
Goal: [INFORM / PERSUADE / DRIVE SIGN-UPS / BUILD AUTHORITY]

Produce a structured brief:

**TOPIC OVERVIEW:** 2-sentence summary of why this topic matters right now

**5 ANGLES TO EXPLORE:**
1. [angle + why it's interesting]
... (repeat)

**KEY INSIGHTS:** (3 data points or expert perspectives — use real stats if available via search, otherwise note [STAT NEEDED])

**OUTLINE:**
- Introduction + hook
- Section 1: ...
- Section 2: ...
- Section 3: ...
- Conclusion + CTA

**BEST HOOK SENTENCE:** (the single most compelling opening line for this post)

Label this output: [BRIEF]
Long-Form Post Draft Prompt
Using the research brief above, write the full long-form post.

Requirements:
- Length: 700–1000 words
- Format: LinkedIn-ready (short paragraphs, line breaks for readability, no markdown headers in the final text — use bold sparingly)
- Structure: strong hook → context → 3 main insights → practical takeaway → CTA
- Voice: match the brand guidelines uploaded to this Project
- End with a question to drive comments: "What has been your experience with [topic]?"

Do not repeat the brief — write the actual post.

Label this output: [DRAFT]
Repurpose Prompt (3 Social Variants)
Using the long-form post above, produce 3 social media variants in one response:

---
VARIANT 1 — LinkedIn Teaser
- Opening hook (1 bold sentence)
- 5 bullet points (the most shareable insights from the post)
- 1 closing CTA sentence
- 3 relevant hashtags
- Max 300 words

---
VARIANT 2 — Email Newsletter Snippet
- Subject line (compelling, no clickbait)
- 2-paragraph plain-text excerpt from the post
- "Read the full article →" CTA with placeholder link
- Max 200 words

---
VARIANT 3 — Twitter/X Thread
- Tweet 1: hook (the most provocative insight — max 280 chars)
- Tweets 2–6: one insight per tweet, numbered (2/6, 3/6 etc.)
- Final tweet: CTA + link placeholder
- Each tweet under 280 characters

Label this output: [SOCIAL VARIANTS]
Content Calendar Artifact Prompt
Generate a self-contained HTML Artifact: a Content Calendar for the next 4 weeks.

Topics planned:
[PASTE YOUR TOPIC LIST HERE — one per line with target publish date]

Each item should be a card showing:
- Topic title
- Target publish date
- Status badge (Draft / Review / Published) — clickable to cycle through states
- Content type (LinkedIn Post / Blog / Email / Thread)

Layout: a week-by-week grid (4 columns = 4 weeks)
Colour: #1E40AF primary
Make status badges clickable — clicking cycles Draft → Review → Published → Draft
Include a summary at the top: total pieces, # in each status

Fully self-contained HTML — no external dependencies.
