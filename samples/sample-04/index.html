Personal Finance Copilot

People upload bank statements and have no idea where their money goes. Manually categorising hundreds of transactions takes hours, and spending patterns stay invisible. This capstone ingests your CSV statement, has Claude categorise every transaction automatically, generates an interactive spending dashboard, and lets you chat naturally about your finances — all using Claude.ai with no API key.

No API key required. This solution is built entirely on your Claude.ai Pro or Team subscription — using file upload, Artifacts, Projects, and conversational Q&A. Your data stays in your Claude.ai account.
Privacy note: Your bank statement CSV is uploaded directly to your personal Claude.ai conversation — it is not sent to any third-party server. Claude.ai's privacy policy applies. For maximum privacy, anonymise merchant names before uploading if preferred.

Architecture — Claude.ai Features Map

Every component of the Finance Copilot maps to a Claude.ai subscription feature. No app to install, no database to set up — just Claude.ai and a CSV.

File Upload

Statement Ingestion

Upload your bank statement CSV directly into Claude.ai. Claude reads the raw transaction data — date, description, amount — and processes every row in one go.

Project

Finance Workspace

A Claude.ai Project stores your monthly statements and categorisation rules persistently. Return next month, upload a new CSV, and Claude applies the same rules consistently.

Skill

Finance Analyst Persona

Project Instructions define Claude as your personal finance analyst — with your custom category taxonomy, savings goals, and budget thresholds built in.

Artifact

Spending Dashboard

Claude generates an interactive HTML Artifact with a bar chart by category, top merchants list, income vs. expense summary, and a searchable transaction table — all from your uploaded data.

Cowork

Joint Finances

Share the Project with a partner or financial advisor. Both can ask questions about the categorised data, review spending patterns, and plan budgets together.

MCP

Budget Benchmarks

Connect a web search MCP to pull average spending benchmarks for your location — "average UK household grocery spend" — so Claude can compare your spending to norms.

Build It Step by Step

  1. Prepare Your Bank Statement CSV Export your bank statement as CSV. It needs at minimum 3 columns: date, description, amount. Negative amounts = expenses, positive = income. A sample CSV is included in this repo at samples/sample-04/sample_statement.csv to practice with.
  2. Create the Finance Project Go to claude.ai → Projects → New Project. Name it "Personal Finance Copilot". This workspace will hold all your monthly statements and categorisation rules over time.
  3. Set the Finance Analyst System Prompt In Project Settings → Instructions, paste the Finance Analyst prompt (see below). Customise the categories to match your spending (e.g. add "Kids" or "Pet Care"). Claude will apply these consistently to every statement you upload.
  4. Upload Your Statement & Run Categorisation Start a new Project conversation. Upload your CSV. Paste the Batch Categorisation Prompt. Claude reads every transaction and assigns a category — returning a clean markdown table plus a JSON summary of totals by category.
  5. Generate the Spending Dashboard Artifact After categorisation, paste the Dashboard Artifact Prompt. Claude creates a full interactive spending dashboard — bar chart by category, income vs. expenses summary, top 10 merchants, and a filterable transaction table — running live in your browser.
  6. Ask Natural Language Questions Now use conversational Q&A. Ask "How much did I spend on eating out?" or "What's my savings rate this month?" Claude answers using the categorised data from your uploaded CSV — no formulas, no spreadsheet needed.
  7. Generate a Monthly Insight Report Paste the Monthly Insight Prompt. Claude produces a plain-English financial health summary: top 3 spending observations, one saving opportunity, one budget suggestion — like a conversation with your personal financial advisor.
  8. Compare with Benchmarks via MCP (Optional) If you've connected a web search MCP, ask Claude to look up average spending in your category for your region. Claude will compare your actual spend to the benchmark and flag if you're significantly over or under.

Prompts & Configs

Project Instructions (System Prompt)
You are my Personal Finance Analyst and Copilot.

Your category taxonomy (apply these consistently to every transaction):
- Groceries: supermarkets, food shops
- Eating Out: restaurants, cafes, takeaway, food delivery apps
- Transport: fuel, public transport, Uber/Lyft, parking
- Subscriptions: Netflix, Spotify, gym, SaaS, insurance (recurring)
- Shopping: clothing, electronics, Amazon general, retail
- Health: pharmacy, GP, dentist, fitness
- Utilities: electricity, gas, water, internet, phone bill
- Income: salary deposits, freelance payments, transfers in
- Savings / Transfers: transfers to savings account, investments
- Other: anything that doesn't fit above

Rules:
1. When I upload a CSV, categorise EVERY transaction using the taxonomy above.
2. Return a markdown table: | Date | Description | Amount | Category |
3. After the table, return a JSON summary: { "by_category": { "Groceries": 234.50, ... }, "total_income": 3200, "total_expenses": 2140, "net": 1060 }
4. For conversational Q&A, always cite the specific transactions that support your answer.
5. Never guess at merchant intent — if a merchant name is ambiguous, put it in "Other" and note it.

My savings goal: save at least 20% of income each month. Flag if I'm on track.
Batch Categorisation Prompt
I've uploaded my bank statement CSV for [MONTH YEAR].

Please:
1. Read every row (date, description, amount)
2. Assign a category to each transaction using my taxonomy (defined in Project Instructions)
3. Return a markdown table with columns: Date | Description | Amount | Category
4. After the table, return the JSON summary block: totals by category, total income, total expenses, net savings, savings rate %

Flag any transactions you're uncertain about with ⚠️ and explain why.

Sort the table by date (oldest first).
Spending Dashboard Artifact Prompt
Using the categorised transaction data from this conversation, generate a self-contained HTML Artifact: a Personal Finance Dashboard.

Include:
1. Summary bar: Total Income | Total Expenses | Net Savings | Savings Rate %
2. Bar chart: spending by category (horizontal bars, sorted largest to smallest) — use canvas or inline SVG
3. Top 10 Merchants table: merchant name + total spent
4. Transaction table: all transactions, filterable by category (dropdown filter)
5. Income vs Expense trend: if multiple months are available, show a mini line chart

Styling: #1E40AF primary colour, clean card layout, mobile-friendly
Fully self-contained — embed all data directly in the HTML.
No external CDN or fetch() calls.

Monthly Financial Insight Prompt
Based on the categorised transactions for [MONTH YEAR], give me a plain-English financial health summary.

Format:
**MONTH:** [month year]
**FINANCIAL HEALTH SCORE:** [A/B/C/D] with one-line explanation

**TOP 3 SPENDING OBSERVATIONS:**
1. [observation + specific amount + % of total spend]
2. ...
3. ...

**BIGGEST SAVING OPPORTUNITY:**
[one specific, actionable suggestion with estimated monthly saving]

**BUDGET SUGGESTION FOR NEXT MONTH:**
[one concrete target to set for next month — specific category + specific amount]

**SAVINGS GOAL STATUS:**
[On Track / Off Track] — [savings rate this month] vs. [20% target] — [amount to save next month to catch up]

Keep it under 250 words. Be direct. Don't sugarcoat overspending.
